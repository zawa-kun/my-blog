# ブランチ開発ワークフロー

すべてのコード変更は、`main` ブランチに直接コミットせず、必ず作業用ブランチを作成してから行うこと。

## 手順

### 1. 作業開始前にブランチを確認する

現在のブランチが `main` であることを確認する。

```bash
git branch --show-current
```

### 2. 作業用ブランチを作成する

最新の `main` から新しいブランチを作成してチェックアウトする。  
ブランチ名は `gemini/<作業内容の要約>` の形式とする。

```bash
git checkout -b gemini/<作業内容の要約>
```

**ブランチ名の例:**
- `gemini/add-about-page`
- `gemini/fix-header-layout`
- `gemini/update-blog-styles`

### 3. コード変更を行う

通常通り、コードの変更・追加・削除を実施する。

### 4. 変更をコミットする

変更内容を適切なメッセージとともにコミットする。

```bash
git add .
git commit -m "<変更内容の要約>"
```

### 5. 作業完了後の報告

作業が完了したら、以下の情報をユーザーに報告する:

- 作業ブランチ名
- 変更内容の概要
- `main` ブランチへのマージはユーザーが確認後に行うことを案内する

> [!IMPORTANT]
> `main` ブランチへのマージは絶対に自動で行わないこと。ユーザーの明示的な指示がある場合のみマージを行う。
